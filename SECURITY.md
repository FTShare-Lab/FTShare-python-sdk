# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

- **Do not** open a public GitHub issue for security vulnerabilities.
- Email your report to **liulei@ft.tech** with a description of the issue, steps to reproduce, and potential impact.

We will acknowledge receipt as soon as possible and work with you on a coordinated fix and disclosure.

> Replace the contact email above with your preferred security reporting address before publishing.

## Scope

This policy covers the `ftshare` Python SDK source code in this repository. Vulnerabilities in third-party dependencies should be reported to their respective maintainers.
